# Contributing to DEALS 2.0

## Development Process

1. **Branch Naming**
   - feature/feature-name
   - fix/bug-description
   - docs/documentation-update

2. **Commit Messages**
   - feat: new feature
   - fix: bug fix
   - docs: documentation
   - style: formatting
   - refactor: code restructuring

3. **Pull Requests**
   - Clear description
   - Link to issue
   - Test coverage
   - Documentation updates

### TypeScript
